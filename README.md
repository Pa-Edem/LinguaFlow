Я разрабатываю приложение "Lingua Flow", которое помогает учить иностранные языки.
Это приложение написано на JavaScript, Vue.js + Pinia, с использованием i18n, Firebase, Google Cloud, Stripe.

Разработка приложения находиться на завершающей стадии, но есть еще не реализованая логика. Я бы хотел, чтобы ты проверил код приложения, сделал анализ проекта, может внес предложения по улучшению или оптимизации проекта.

Основная идея приложения заключается в том, что пользователь создаёт диалоги с использованием нейросети.

При создании диалога он указывает тему диалога, слова обязательные для использования в этом диалоге, уровень сложности и количество реплик.
Созданный диалог сохраняется в базе firestore.

Пользователь может использовать любой из созданных им диалогов в различных целях.
Может получить лексико-грамматический анализ данного диалога.
Может прослушать диалог на выбранном для изучения языке.
Может учить этот диалог построчно.
Может проверять произношение.
Может проверять корректность перевода.
Может проверять правильно ли он понимает на слух.

В приложении реализована трехуровневая система пользователей. Обычный пользователь имеет free-план. Пользователь, который оформил платную подписку, имеет pro-план или premium-план.

Управление подпиской осуществляется на странице "профиля пользователя".
На странице "настройки" можно выбрать светлую или тёмную тему приложения, можно выбрать язык приложения (язык интерфейса), также выбрать язык для изучения. Для пользователей с pro-планом и premium-планом есть секция настройки голоса. Пользователь с free-планом использует браузерную озвучку. Пользователь с pro-планом и premium-планом может использовать как озвучку браузера, так и выбрать один из десяти голосов, которые представлены сервисом Google Cloud TTS. Также есть возможность регулирования скорости речи.

На странице "профиль" пользователь имеет возможность выйти из приложения или удалить аккаунт. На странице "просмотр диалога" пользователь имеет возможность удалить данный диалог из базы.

Для того, чтобы пользователи с free-планом почувствовали достоинства платной подписки, реализована система суточных и недельных (накопительных) лимитов на некоторые функции (pro-функции).

### Pro-функции:

- лексико-грамматический анализ диалога,
- проверка правильности произношения,
- проверка правильности перевода.

---

## Объяснение планов и системы лимитов.

### Пользователь с free-планом:

- использует браузерную озвучку,
- каждый день получает 2 генерации,
- неиспользованные генерации накапливаются,
- может генерировать не более 4 диалогов в день,
- может создать не более 10 диалогов в неделю,
- общее количество сохраненных диалогов не может превышать 10.
- каждый день получает возможность использовать 2 pro-функции,
- неиспользованные возможности использовать pro-функции накапливаются,
- получает возможность использовать не более 8 pro-функций в день,
- получает возможность использовать не более 10 pro-функций в неделю.

### Пользователь с pro-планом:

- использует 10 голосов качественной озвучки,
- каждый день получает 10 генерации,
- неиспользованные генерации накапливаются,
- может генерировать не более 15 диалогов в день,
- может создать не более 50 диалогов в неделю,
- общее количество сохраненных диалогов не может превышать 50,
- каждый день получает возможность использовать 20 pro-функции,
- неиспользованные возможности использовать pro-функции накапливаются,
- получает возможность использовать не более 50 pro-функций в день,
- получает возможность использовать не более 100 pro-функций в неделю,
- получение лексико-грамматического анализа диалогов без лимитов.

### Пользователь с premium-планом:

- все, что и pro-план, но без ограничений.

Структура в Firestore
config/limits {
free: {
dailyGenerations: 2, (плюс каждый день)
dailyGenerationsMax: 4, (Максимум создать в день)
weeklyGenerationsCap: 10, (Максимум создать в неделю)
dailyPreview: 2, (плюс PRO-функций каждый день)
dailyPreviewMax: 8, (Максимум PRO-функций в день)
weeklyPreviewCap: 20, (Максимум PRO-функций в неделю)
totalDialogs: 10, (Максимум диалогов сохранено)
unlimitedAnalysis: false
},

pro: {
dailyGenerations: 10,
dailyGenerationsMax: 15,
weeklyGenerationsCap: 50,
dailyPreview: 20,
dailyPreviewMax: 50,
weeklyPreviewCap: 100,
totalDialogs: 50,
unlimitedAnalysis: true,
},

premium: {
unlimited: true
}
}
